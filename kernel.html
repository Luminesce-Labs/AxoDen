The governing invariant

The CMRI Bound: Independence as a design constraint

The Correlation-Minimised Redundancy Invariant is a normative requirement, not a statistical measurement. A system either satisfies it by construction or it does not. No test suite can substitute for this guarantee.

CMRI := supω∈Ω maxi<j ρ̄ij(ω) ≤ 1 − ISmin + εA

Where ρ̄ij(ω) is the trust-failure correlation between any two redundant AI components at operating point ω, ISmin is the minimum independence score threshold, defined per ASIL-M deployment level, and εA is the architectural tolerance. Enforced by structural CFS constraints — not measured post-hoc. Every existing framework assumes this bound holds. AxoDen is the first to prove it.

Mathematical foundations

Three compositional pillars

EBDP, EFI, and CFS are independently certifiable and compose algebraically: local constraints become global guarantees. When a component changes, only local constraints require re-verification.

EBDP

Entropy-Bounded Data Pipelines

H(Xk | Xk-1) ≤ Hmaxk

At every pipeline stage k, uncertainty accumulation is bounded from above. Entropy cannot silently compound across stages. A breach is a detectable invariant violation, not a statistical anomaly. Applied to sensor fusion, RAG pipelines, multi-hop reasoning chains, and RF signal ingestion. DOI: 10.5281/zenodo.17709557

EFI

Entropy-Frugal Intelligence

I(Xk ; Mk) ≤ εk

The mutual information a model may introduce at stage k — not derivable from evidential inputs — is bounded by εk. Model-injected hallucination is structurally limited, not behaviourally monitored. The bound is derivable from the Evidence Object chain without inspecting model internals.

CFS

Coprime-Factor Security

ISatt ≥ ISmin    v2(hA ⊕ hB) ≤ kmax

Independence engineered across four planes: cryptographic roots, vendor supply chains, operational domains, and hardware platforms. The 2-adic distance metric v2(hA ⊕ hB) provides bit-level structural correlation detection with no dependency on behavioural observation. DOI: 10.5281/zenodo.17690358

Runtime enforcement

Three runtime control modules

The Kernel enforces its invariants at runtime through three coupled modules. All state transitions are deterministic, ledger-backed, and cryptographically auditable. There is no ambiguity in adversarial conditions.

ARV

Adaptive Runtime Verifier

Autonomous Recovery Valve

Deterministic fail-closed policy enforcement via a named precedence chain. When multiple rules apply, the highest-precedence rule governs — no ambiguity under adversarial conditions. Evaluates intents against topological geofence and quorum-state bounds in O(1) time with respect to intent count, ensuring bounded authorisation latency regardless of queue depth.

MQ

Monitoring Quad

Four explicit operating states — no implicit state

State coupling and degradation management across four explicit operating states. Every AI component governed by the Kernel is in exactly one state at all times. Transitions are deterministic and logged as Canonical Attestation Records (CARs). No silent state drift.

Certificate-based topology verification

Maintains the ER-topo.cert rotation system certificate for the current evidence graph. Verifies topology in O(E) time without recomputing the embedding. The verification key is decoupled from the construction key — any independent party can verify the topology certificate without access to the construction process or trust in the constructor.

ER-topo.cert: O(E) certificate-based topology verification

The same asymmetry that makes public-key cryptography operationally feasible, applied to AI evidence graph topology. Construction is expensive, done once by a trusted producer. Verification is linear time, done by any independent party. Prior art established and DOI-registered February 2026.

Construction-verification separation

A trusted producer constructs the rotation system certificate for the evidence graph (O(E log g) where g is genus). Any verifier checks the certificate in O(E) time by traversing the face-cycle structure. The verifier does not recompute the embedding. No equivalent architecture exists in distributed systems or formal verification literature.

ZIH — TLA+ Verified — Universal failure class

Zombie Intent Hazard

A universal structural failure class in any system where AI decisions are queued, buffered, or persisted before physical execution. Not a cybersecurity vulnerability. Applies equally to power grids, autonomous vehicles, medical devices, industrial control systems, and autonomous drones. A stale intent executes under conditions for which it was never valid. God Key — Single Root Silent Invalidation

An audit architecture with a single signing root allows silent, retroactive invalidation of the entire audit chain. From deployment to certification

The certification pathway

These are the steps from initial deployment to certification-grade evidence submission. AxoDen does not invent a new certification route. It produces evidence for existing ones.

01

Select ASIL-M conformance profile

Choose L1 through L4 based on consequence severity and applicable standards. The profile determines which artefact categories are required and the ISmin threshold the design must satisfy.

Deliverable: conformance profile selection document

02

Configure EBDP and EFI bounds

Define Hmaxk and εk for each pipeline stage based on domain physics and information flow requirements. The Kernel enforces these bounds at runtime and logs every violation as a CAR.

Deliverable: entropy budget document; stage-level EFI ceiling specification

03

Commission CFS independence analysis

Document coprime monitor root selection across the four independence planes. Produce ISatt measurement demonstrating compliance with ISmin at the chosen ASIL-M level.

Deliverable: CFS independence report; ER-cfs audit artefacts; CMRI bound verification

04

Generate and verify ER-topo.cert certificate

Construct the rotation system certificate for the evidence graph topology. Any independent party can verify it in O(E) time. This is the structural safety proof — the certificate that distinguishes AxoDen from every preceding framework.

Deliverable: ER-topo.cert certificate file; verifier test results; topological slack S measurement

05

Commission TAPL policy and produce CAR baseline

Formalise governing policies in TAPL. Run the system under the target workload profile and produce the baseline CAR chain demonstrating deterministic behaviour. Bit-exact replay verification confirms determinism across runs and platforms.

Deliverable: TAPL policy corpus; CAR chain baseline; replay verification log

06

Submit artefact bundle to assessor

The assembled artefact bundle maps directly onto evidence categories existing safety assessors already evaluate under DO-178C, IEC 61508, or DEF STAN 00-56. No new certification path. No novel evidence category. The assessor already knows what to do with this.

Deliverable: complete ASIL-M artefact bundle; assessor-ready evidence package
